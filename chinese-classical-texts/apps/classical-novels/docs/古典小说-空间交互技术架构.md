# 古典小说 · 空间交互技术架构（沉浸地图 / NPC 对话）

> 大观园、西门府等**可交互实景地图**与 **NPC 点击对话**的技术宪法。  
> 一句话结论：**2D 优先，运行时跑在现有 Astro 项目内（React 岛 + PixiJS），美术资产用独立工具产出后投放——不做独立游戏再 iframe 嵌入。**

---

## 1. 关键决策

| 问题 | 结论 | 理由 |
|------|------|------|
| 2D 还是 3D | **2D 先行且为长期主力**，3D 后期单点沉浸 | 建筑图/背景/方位距离/NPC 点击 2D 全能满足，成本仅 3D 零头 |
| 浏览器内 vs 独立项目 | **浏览器内 React 岛**（Pixi），非 iframe | 直接复用 `locations/characters/events` 数据；点 NPC 可跳 wiki 词条；同一构建链 |
| NPC「随机对话」 | **从角色原文台词池随机抽**（带回目出处） | 守本库铁律「情节须有出处」；非 LLM 现编 |
| 美术来源 | AI 生成 / 考证版画扫描，落 `public/`，**缺失回退矢量占位** | 框架与资产解耦，换图不改码 |

---

## 2. 分层架构

同一套内容数据（`src/content/`、`src/data/`）驱动多种渲染器，按用途选层：

| 层 | 渲染器 | 路由 | 状态 |
|----|--------|------|------|
| **地图索引** | Astro 卡片 | `/[book]/maps` | ✅ 红楼五卡汇总 |
| 逻辑图 | ECharts | `/honglou/map`（36 节点）· `/manor`（26）· `/capital`（26） | ✅ 已上线 |
| 数字文旅 | HTML + 底图标注 | `/honglou/digital-tour` · Archviz 鸟瞰 + 36 pin | ✅ 已上线 |
| **实景 2.5D** | **PixiJS v8 React 岛** | `/[book]/scene` · 全园 36 栋 + 距离 + NPC | ✅ 已上线（红楼） |
| 距离/地理 | Leaflet（Simple CRS） | 强调步数/比例的可缩放地图 | ⬜ 规划（A10） |
| 3D 场景 | React Three Fiber + glTF | 单院沉浸（如 `/honglou/scene3d`） | ⬜ 规划（A11） |

> 资产生产用**独立工具**：Tiled（埋建筑边界/NPC坐标/距离锚点）、Blender（低模+法线烘焙 glTF）、PS/AI（背景与立绘）。产物导出到 `public/<book>/...` 由岛加载，**不部署成独立项目**。

## 3. 数据契约

### 3.1 场景配置 `src/data/<书>.garden_scene*.json`

- **全园 manifest**：`红楼梦.garden_scene_full.json`（36 栋 · 与 `garden_map.json` 同源 `seed_garden_coords`）→ `/honglou/scene` 加载。
- **一角 pilot**：`红楼梦.garden_scene.json`（3 栋示意）保留作契约样例。
- **数字文旅 pin**：`红楼梦.garden_digital_tour_pins.json`（36 处 xPct/yPct）→ `/honglou/digital-tour`。

```jsonc
{
  "title", "subtitle", "width", "height",
  "background": "/honglou/scene/garden-bg.png",  // 可选，缺失回退渐变+水系矢量
  "px_per_step": 6,            // 距离换算：像素→步（inference 示意）
  "scale_note": "...",         // 标注非测绘尺度
  "buildings": [{ "id","name","plaque","zone","x","y","w","h","sprite","link","note" }],
  "npcs":      [{ "id","name","at","x","y","sprite","link","tint" }],
  "paths":     [{ "from","to" }]   // 仅连线，距离按建筑中心欧氏距离/px_per_step 计算
}
```

### 3.2 台词池 `src/data/<书>.dialogues.json`

```jsonc
{
  "pools": {
    "林黛玉": [{ "text": "原文逐字…", "chapter": 27, "source": "chapters/红楼梦/027.md" }]
  }
}
```

- 每条**必须可回原文核对**；新增须过 `/guard`。
- 点 NPC → 从其 pool 随机抽一句，气泡展示「原话 + 第N回 + 词条链接 + 再说一句」。

## 4. 组件与交互（`src/components/GardenScene.tsx`）

- **Pixi 画布**负责渲染 + 命中检测；**HTML overlay** 负责面板/对话（Tailwind，可链 wiki）。
- 建筑/立绘**按原图宽高比落位**（底部对齐），不强制拉伸；贴图缺失回退矢量占位（歇山顶/台基/人物剪影）。
- 交互：点建筑→方位距离面板（到各建筑「约 N 步」）+ 词条链接；点 NPC→随机原文对话；滚轮缩放、拖拽平移、重置视图。
- 接入：`features` 含 `scene` / `garden` / `manor` / `capital` → 顶栏「空间地图」→ `/honglou/maps` 五卡 → 各子路由（`scene.astro` 读 `garden_scene_full.json`）。

## 5. 美术资产流水线

```
AI 生成 / 版画扫描  →  cutout 抠图（如需）  →  public/<book>/scene/  →  岛自动加载
```

- **投放即生效**：按 `README` 约定文件名放入 `public/<book>/scene/`，无需改码。
- **「假透明」修复**：AI 常把棋盘格画成实色 RGB（无 alpha）。`scripts/cutout_scene_assets.py` 从图像**边缘 flood-fill** 把近中性亮色背景抠成真透明——建筑内墙因不与边缘连通而保留。
- 已落地（红楼）：`public/honglou/scene/` 6 张绢本工笔资产（背景 + 潇湘馆/怡红院/沁芳亭 + 黛玉/宝玉立绘）。

## 6. 资料来源策略（图纸 / 美术）

详见两份资料目录页，本架构只记**取用次序**与**可信分层**：

- 红楼：[[大观园建筑图纸与美术资料来源]]
- 金瓶：[[西门府建筑图纸与美术资料来源]]

| 分层 | 含义 | 用法 |
|------|------|------|
| **内部·可核** | 原著正文（如第17回间数/抱厦） | 可作建筑参数硬依据，标回目 |
| **内部·可用** | 本库已存扫描件（金瓶崇祯本绣像 `raw/editions/jinpingmei/JPM_Congzhen/image/`） | 可直接抠图作美术资产 |
| **外部·待核** | 学者平面图、馆藏画作、营造典籍 | 仅作美术/工程参考，**不当作原意佐证** |

**坐标权威（现网）**：仍为 **南北中轴说 inference**（`seed_garden_coords` / `gardenSceneCoords.ts`）。scan 图、杨乃济平面图 **不参与落位**；若迁移须单独架构决策（路线图 A12）。

落地次序：① 考证平面图作 2D 底座（杨乃济 / 白维国）→ ② 叠原著参数 → ③ 工笔/版画定美术风格 → ④《清式营造则例》《园冶》校 3D 比例。
