# 关系图谱 · 前端布局规范（防复发）

> 三书共用 `/[book]/graph`，组件 `RelationGraph.tsx` + 布局 `GraphLayout.astro`。  
> **现象**：页面空白、一直「加载关系图谱…」、有工具栏但无节点——多为画布高度为 0，不是 JSON 数据丢失。

---

## 根因

ECharts 在 `<canvas>` 上绘图，**容器必须有 ≥16px 的明确高度**。若高度为 0：

1. `RelationGraph` 内 `mountChart` 会等待容器尺寸；
2. 旧实现用 `requestAnimationFrame` 无限重试 → 永远空白；
3. 用户误以为「关系数据没了」，其实 `src/data/<书>.relations.json` 仍在。

### 为何反复踩坑

| 变更 | 风险 |
|------|------|
| 顶栏加 `TopModuleNav`（第二行模块卡） | 实际顶栏高度 > 固定 `7.5rem` |
| 全站 typography / `global.css` 改顶栏 padding | 顶栏变高，原 `--graph-chrome` 不准 |
| 去掉 `RelationGraph` 内联 `height`、只靠 `height: 100%` | 父级高度链一断，画布即 0 |
| 把整份 relations JSON 塞进 `astro-island` props | 岛屿过大、hydration 失败（已改为组件内 `relationGraphForSlug`） |

历史提交：`2d30397`（组件内加载 + chrome 变量）→ `d13b593`（grid 壳但去掉显式高度）→ 回归空白。

---

## 必须保留的三层防护

### 1. CSS 变量 `--graph-chrome`

表示**顶栏占用高度**（含 TopModuleNav）。正文区高度：

```css
calc(100dvh - var(--graph-chrome, 10.5rem))
```

- 定义于 `GraphLayout.astro` 的 `<body class="graph-shell">` 默认 `--graph-chrome: 10.5rem`
- **动态同步**：内联脚本读取 `.graph-topbar.offsetHeight` 写回 `--graph-chrome`（`resize` / `load` / `fonts.ready` / 顶栏 **ResizeObserver** 时再测）

### 2. 布局链

```text
.graph-shell          grid: auto 1fr; height: 100dvh
  .graph-topbar       auto
  .graph-main         height: calc(100dvh - var(--graph-chrome)); min-height: 0
    .graph-explorer   height/min-height 同上（RelationGraph 根节点）
      chartRef div    absolute inset-0; ECharts init 目标
```

**禁止**只依赖 `height: 100%` 而不给 `.graph-main` / `.graph-explorer` 显式 `calc(100dvh - var(--graph-chrome))`。

### 3. 数据加载

- **禁止**经 `astro-island` props 传递整份 `*.relations.json`（可达 ~100KB+）
- **必须**在 `RelationGraph` 内 `relationGraphForSlug(bookSlug)` 静态 import
- island 只传 `bookSlug: string`

### 4. ECharts 初始化

- 用 **ResizeObserver** 观察 `.graph-explorer` 与 canvas 容器，有尺寸后再 `echarts.init`
- 高度仍不足时，用 **有上限的 `requestAnimationFrame` 重试**（≤240 帧），勿无限循环
- `GraphLayout` 在 `--graph-chrome` 更新后派发 `graph-chrome-sync`，组件监听并重试 mount / resize（**重置 rAF 计数、清除 chartError 遮罩**）
- 超时（~240 次仍 <16px）显示 `chartError`，勿无声失败

## 快速排查

| 症状 | 查什么 |
|------|--------|
| 一直转圈 | 浏览器 Network：`RelationGraph.*.js` 是否 200；Console 是否有 hydration 错误 |
| 空白 + 顶栏正常 | DevTools 选中 canvas 父级，看 `clientHeight` 是否为 0 |
| 提示「待生成」 | 跑 `python scripts/build_relations.py <书>`，查 `src/data/<书>.relations.json` |
| 仅 dev 坏 | `npm run dev:clean`（`.astro` 锁）；确认端口唯一 |
