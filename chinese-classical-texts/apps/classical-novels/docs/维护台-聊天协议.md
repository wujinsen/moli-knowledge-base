# 维护台 · 聊天协议（草案 v0.1）

> 状态：草案 · 2026-06-16  
> 目标：在站点内用 **上下文聊天 + 变更卡片** 驱动知识库自我进化，替代「切到 Cursor 对话框改信物/服饰」。  
> 关联：[`AGENTS.md`](../AGENTS.md) · [`维护台-使用说明.md`](维护台-使用说明.md) · [`红楼梦-知识图谱架构.md`](红楼梦-知识图谱架构.md) · [`2026智能知识库演进路线.md`](../../../docs/2026智能知识库演进路线.md)

---

## 1. 设计原则

| 原则 | 说明 |
|------|------|
| **聊天 = 意图层** | 用户自然语言；LLM 解析为结构化 `intent` + `tool_calls` |
| **写盘 = 卡片层** | 任何改 `src/content/` 或 `src/data/` 必须先 `propose_patch`，用户点 **应用** 后才 `apply_patch` |
| **上下文自动注入** | 打开维护侧栏时，当前页实体 / 回目 / lint 信号随 `MaintenanceContext` 下发，用户不必 @ 文件 |
| **chapters 只读** | Tool 可读 `chapters/`，禁止 write；情节结论须带 `source.chapter` |
| **AGENTS.md 为宪法** | Agent system prompt 固定引用；关系词表、回填规则、log 格式不可绕过 |
| **进化闭环** | `lint` 发现 → 聊天推送待办 → 提案 → 应用 → `guard` → `graph`/snapshot → UI 刷新 |

## 3. MaintenanceContext（页面上下文）

打开维护侧栏时，前端 `POST /api/studio/sessions` 携带：

```json
{
  "book": "红楼梦",
  "bookSlug": "honglou",
  "locale": "zh-CN",
  "page": {
    "kind": "character",
    "route": "/honglou/c/贾迎春",
    "entityId": "贾迎春",
    "entityType": "character"
  },
  "viewer": {
    "role": "maintainer"
  }
}
```

Orchestrator 响应 `sessionId`，并附 **服务端组装的完整上下文**（`MaintenanceContext`）：

```json
{
  "sessionId": "sess_01J…",
  "version": "0.1",
  "book": "红楼梦",
  "bookSlug": "honglou",
  "rulesRef": "apps/classical-novels/AGENTS.md",
  "page": {
    "kind": "character",
    "entityId": "贾迎春",
    "entityType": "character",
    "name": "贾迎春",
    "route": "/honglou/c/贾迎春"
  },
  "entity": {
    "path": "src/content/characters/红楼梦/贾迎春.md",
    "frontmatter": {
      "id": "贾迎春",
      "关键物品": ["累丝金凤"],
      "服饰": [],
      "first_appear": "第3回",
      "summary": "…"
    },
    "plotBullets": 4,
    "relationsCount": 8
  },
  "items": {
    "keyItems": [
      {
        "id": "累丝金凤",
        "kind": "costume",
        "wikiPath": "src/content/costumes/红楼梦/累丝金凤.md",
        "wearer": "王熙凤",
        "first_appear": "第21回",
        "linked": true,
        "issues": ["wearer_mismatch_ch73"]
      }
    ],
    "costumes": [],
    "crosslinksOccupant": ["迎春出嫁"],
    "missingFromCrosslinks": ["累丝金凤"]
  },
  "chapters": {
    "suggested": [73, 74, 80],
    "withItems": [
      { "n": 73, "items": ["累丝金凤"], "summary": "…" }
    ]
  },
  "lintHints": [
    {
      "code": "crosslinks_missing_item",
      "severity": "warn",
      "message": "occupant_items 缺 累丝金凤，但 frontmatter 关键物品已列"
    }
  ],
  "allowedIntents": [
    "fix_key_item",
    "fix_costume",
    "add_plot_bullet",
    "query",
    "run_guard",
    "run_lint"
  ]
}
```

### 3.1 字段说明

| 字段 | 来源 |
|------|------|
| `entity.frontmatter` | 解析人物 md |
| `items.*` | frontmatter + `hongloumeng.item_ids.json` + `*.crosslinks.json` + 名物页 |
| `chapters.suggested` | `first_appear`、关键情节回目、crosslinks、用户当前阅读器 URL |
| `lintHints` | 轻量规则（非完整 lint_kb）；完整 lint 走 `run_lint` tool |
| `issues[]` | 名物页 `wearer` 与情节回目不匹配等启发式 |

### 3.2 JSON Schema

机器可读定义见 [`schemas/studio/maintenance-context.schema.json`](../schemas/studio/maintenance-context.schema.json)。

## 4. 聊天消息协议

### 4.1 客户端 → 服务端

```http
POST /api/studio/sessions/{sessionId}/messages
Content-Type: application/json

{
  "text": "第73回乳母赌金凤，把信物链补全",
  "clientMessageId": "msg_client_001"
}
```

可选 **快捷意图**（按钮触发，等价于带结构化 hint）：

```json
{
  "text": "",
  "intentHint": {
    "type": "fix_key_item",
    "params": { "itemId": "累丝金凤", "chapter": 73 }
  }
}
```

### 4.2 服务端 → 客户端（SSE 事件流）

事件类型：

| event |  payload 要点 |
|-------|----------------|
| `message.delta` | `{ "text": "对照第73回…" }` 流式正文 |
| `message.done` | `{ "messageId", "role": "assistant" }` |
| `tool.start` | `{ "tool": "read_chapter", "args": { "chapter": 73 } }` |
| `tool.done` | `{ "tool", "summary": "命中「累丝金凤」3处" }` |
| `proposal.ready` | **`PatchProposal` 完整对象**（见 §5） |
| `guard.result` | `{ "verified": 2, "unverified": 0, "items": [...] }` |
| `error` | `{ "code", "message" }` |

**规则**：若本轮会改库，assistant 必须在同一 turn 内发出 **至少一个** `proposal.ready`；禁止仅文字声称「已改好」。

## 5. PatchProposal（propose_patch 响应）

### 5.1 结构

```json
{
  "proposalId": "prop_01J…",
  "sessionId": "sess_01J…",
  "status": "pending",
  "intent": "fix_key_item",
  "title": "贾迎春 · 补全累丝金凤信物链",
  "summary": "对照第73回，更新名物页 wearer、crosslinks、关键情节各 1 处。",
  "sources": [
    {
      "chapter": 73,
      "path": "src/content/chapters/红楼梦/073.md",
      "edition": "zhiben",
      "excerpt": "…累丝金凤…",
      "readUrl": "/honglou/read/zhiben/73"
    }
  ],
  "patches": [
    {
      "path": "src/content/costumes/红楼梦/累丝金凤.md",
      "operation": "update",
      "hunkSummary": "wearer 增第73回迎春乳母赌局；appear_in +73",
      "diff": "--- a/…\n+++ b/…\n@@ …"
    },
    {
      "path": "src/content/characters/红楼梦/贾迎春.md",
      "operation": "update",
      "hunkSummary": "关键情节 +1（第73回）",
      "diff": "…"
    },
    {
      "path": "src/data/hongloumeng.crosslinks.json",
      "operation": "update",
      "hunkSummary": "occupant_items.贾迎春 += 累丝金凤",
      "diff": "…"
    }
  ],
  "postApply": {
    "scripts": [
      "python scripts/build_crosslinks.py 红楼梦",
      "python scripts/build_hlm_bestiary_json.py",
      "python scripts/trust_guard.py 红楼梦 --scope 贾迎春,累丝金凤",
      "python scripts/build_content_snapshots.py --write"
    ],
    "logEntry": "## [2026-06-16] studio | 贾迎春 累丝金凤 crosslinks + wearer"
  },
  "guardPreview": {
    "status": "pass",
    "checked": 2,
    "unverified": []
  },
  "actions": {
    "canApply": true,
    "canApplyPartial": true,
    "canEdit": true,
    "canDiscard": true
  }
}
```

### 5.2 patch 操作类型

| operation | 说明 |
|-----------|------|
| `create` | 新建名物页等 |
| `update` | unified diff 应用 |
| `delete` | MVP 禁用（需 maintainer 显式 flag） |

### 5.3 意图枚举（MVP）

| intent | 触发示例 | 主要改动文件 |
|--------|----------|--------------|
| `fix_key_item` | 补/改信物 | `characters/*.md` `关键物品[]`、名物页、`crosslinks`、情节 |
| `fix_costume` | 改服饰 | `characters/*.md` `服饰[]`、名物页 `wearer` |
| `add_plot_bullet` | 补可核情节 | `characters/*.md` 正文 |
| `query` | 只问不改 | 无 patch；可选 `topic.fillSuggestion` |
| `run_lint` / `run_guard` | 体检/校验 | 无 patch；返回报告 |

### 5.4 JSON Schema

[`schemas/studio/patch-proposal.schema.json`](../schemas/studio/patch-proposal.schema.json)

## 6. Agent Tools（LLM 可调）

Orchestrator 向 LLM 暴露 OpenAI-style tools：

| tool | 参数 | 副作用 |
|------|------|--------|
| `read_chapter` | `{ book, chapter, edition? }` | 只读 |
| `read_entity` | `{ book, kind, id }` | 只读 |
| `read_item` | `{ book, itemId }` | 只读 |
| `search_chapter` | `{ book, terms[] }` | 只读，锚词检索 |
| `propose_patch` | `{ intent, title, patches[], sources[], postApply }` | 写 proposal 记录，**不写 git** |
| `run_guard_preview` | `{ book, entityIds[], itemIds[] }` | 只读校验 |
| `suggest_topic_fill` | `{ title, derived_from[], summary }` | 仅 query 意图；生成 topic 草稿进 proposal |

**禁止 tools**：`write_file`、`git_commit`、`edit_chapter_body`。

### 6.1 fix_key_item 提示词骨架（system 摘录）

```text
你是红楼梦知识库维护者。当前上下文见 MaintenanceContext JSON。
任务：按用户意图修正「关键物品/信物」链。
铁律：
- chapters/ 只读；情节须有 source.chapter
- 关系 type 仅用 AGENTS.md 词表
- 名物 id 须与文件名一致；新物走 costumes|dishes|medicines|customs
- 改 crosslinks 后 postApply 须含 build_crosslinks.py
完成后必须调用 propose_patch，不得声称已写入。
```

## 7. REST API（Orchestrator）

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/studio/sessions` | 创建会话 + 返回 MaintenanceContext |

```json
  ],
  "skipPostApply": false
}
```

```json
  "status": "running",
```

## 9. 自我进化闭环（lint → 聊天）

### 9.1 推送规则（Phase 2 · 已落地 MVP）

`GET /api/studio/todos/{bookSlug}` 聚合 crosslinks 缺口、ingest 抽样、低密度人物；维护台右上角 **待办** 按钮一点即带 `intentHint` 开聊。

构建时可选生成 `public/data/studio-todos.json`（静态部署用）；dev 下直接调 API。

```json
{
  "book": "红楼梦",
  "generatedAt": "2026-06-16T12:00:00Z",
  "todos": [
    {
      "id": "todo_crosslinks_迎春_累丝金凤",
      "kind": "fix_key_item",
      "entityId": "贾迎春",
      "message": "关键物品已列累丝金凤，crosslinks 未链",
      "suggestedPrompt": "补全贾迎春与累丝金凤的 crosslinks 与名物页第73回论据"
    }
  ]
}
```

UI：**待办 1 条** → 点开即带 `intentHint` 创建 session。

### 9.2 与 AGENTS.md 命令映射

| AGENTS 命令 | Studio 入口 |
|-------------|-------------|
| `/ingest` | Studio **摄取** Tab + 阅读器 dev「维护本回」+ `GET /api/studio/ingest/{bookSlug}/{chapter}` + `ingest_chapter` intent |
| `/query` | 聊天默认；`suggest_topic_fill` |
| `/lint` | Studio 批处理 Tab **`/lint`** + `GET /api/studio/lint/{bookSlug}` |
| `/dream` | Studio **`/dream`** · tier 目录 + dry-run 预览 · `POST` 应用 + postApply（仅 honglou） |
| `/graph` | Studio 批处理 **`/graph`** · `GET` 预览 · `POST` 写盘 `*.relations.json` |
| `/guard` | Studio **`/guard`** · `GET /api/studio/guard/{bookSlug}` |

## 10. 安全与权限

| 项 | 要求 |
|----|------|
| 写操作 | `viewer.role === maintainer'`；读者模式仅 `query` + `run_lint` |
| CORS | 仅 dev origin + 维护子域 |
| Rate limit | 每 session 10 proposals / hour |
| Audit | `src/content/<书>.log.md` 每条 apply 必记 `studio \| …` |
| Rollback | Apply 前 `git stash` 或 proposal 快照；MVP 可手动 git revert |

## 11. MVP 验收（信物 / 服饰）

1. 打开 `/honglou/c/贾迎春`，点 **维护**  
2. 输入：「第73回乳母赌金凤，补 crosslinks 和名物页 wearer」  
3. 收到 `PatchProposal`，guard preview pass  
4. 点 **应用全部** → postApply 成功 → 图鉴信物 chip 可点 → 累丝金凤页含第73回  
5. 打开 `/honglou/c/林黛玉`，输入：「宫制堆纱花从服饰移到信物」→ diff 正确 → 应用后人物页字段更新
