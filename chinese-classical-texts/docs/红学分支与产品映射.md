## 红学是什么

**红学（Redology）** 是专门研究《红楼梦》的综合性学问。自清代中后期诞生以来，因小说未完稿且蕴含极丰富的社会文化信息，研究早已超越文学评论，成为一门庞大显学。

红学家的工作像**法医、侦探与文化学者的结合**：有人比对抄本错字（版本学），有人翻皇家档案（曹学），有人研究大观园营建（名物学），有人剖析人性与思想（文学思想）。这些维度共同支撑「红学」大厦——本项目的 `classical-novels` 应**按分支组织模块**，而非只做「人物关系图 + 阅读器」。

## 一、考证派四大支柱 → 核心底座

考证派解决《红楼梦》的「身世之谜」，对应项目 **raw 层 + 版本/溯源能力**。

| 红学分支 | 研究对象 | 项目映射 | 数据层 | 功能 ID |
|----------|----------|----------|--------|---------|
| **版本学** | 甲戌、庚辰、甲辰等脂本；程甲、程乙等刻本；源流与文字真伪 | 多版本 `edition` 并列；异文 `variants`；对勘 UI | `raw/daizhige/`、`raw/editions/`；chapter `edition`+`source` | H-10, H-11, B-16, B-15 |
| **曹学** | 曹雪芹身世、江宁织造曹家兴衰；贾府现实原型 | 作者/家族 wiki 页；历史事件 ↔ 小说情节对照 | `wiki/topics/`、`external/` 史料摘要 | H-12, B-15（引文溯源） |
| **脂学** | 脂砚斋、畸笏叟等批语；创作过程与后四十回剧透 | 批语作为 `annotation` 实体；与回目/段落锚定 | 脂评本 raw（待补）；`chapter` 扩展 `annotations[]` | B-15, H-10, B-17 |
| **探佚学** | 判词、伏笔、脂批 → 原构思结局与人物命运 | **假说卡**（inference）与 **事实**（fact）分层；多结局分支图 | `wiki/` 探佚专题；`contradicts` / 虚线边 | B-13, B-17, H-11 |

**工程原则**（与技术方案一致）：

- raw 只读，多版本并列，不在 raw 层合并。
- 展示用通行本（如殆知阁程高本 120 回），研究用可对勘脂本。
- 探佚、曹学推论一律标「诠释」，挂证据链，Trust Guard 校验事实边。

## 二、文学与思想艺术 → wiki + 读者层

将《红楼梦》回归文学作品本质，对应 **wiki 维护层 + 读者/师生功能**。

| 研究方向 | 内容 | 项目映射 | 功能 ID |
|----------|------|----------|---------|
| **人物与叙事** | 宝黛钗悲剧、人际网、「草蛇灰线」多线结构 | `character` / `relation` / 时序图谱；因果链图 | R-05~06, R-10, B-11~12, T-05 |
| **思想与哲学** | 儒释道、对宗法/科举/礼教的批判 | `topic` 主题页；思想词条图谱 | T-01, T-06, wiki `topics/` |
| **诗词曲赋** | 「按头制帽」、判词与情节暗示 | `poem` / `omen` / `symbol`（见 [`红楼梦-知识图谱架构.md`](红楼梦-知识图谱架构.md) 维度④）；诗词—人物—回目链 | B-02, R-02, T-08~10 |

## 三、百科与名物考证（文化红学）→ 垂直纵切

《红楼梦》被誉为「封建社会百科全书」，对应 **结构化实体库 + 多维检索 + 纵切研究样板**。

| 研究方向 | 内容 | 项目映射 | 纵切文档 / 功能 |
|----------|------|----------|------------------|
| **建筑与园林** | 大观园方位、布局、植物；复原图纸 | `location` 扩展；**2D 总图 + 3D 漫游** | R-08b, P5 |
| **医药与饮食** | 冷香丸、诊脉、茄鲞等 | `dish` / `banquet` / `ingredient` / 药膳 | [`红楼梦-饮食纵切研究.md`](红楼梦-饮食纵切研究.md) |
| **服饰与民俗** | 穿着、首饰、节庆、婚丧、货币物价、奴婢 | `garment` / `custom` / 物价节点 | B-07, B-08（金瓶梅同类） |
| **其他名物** | 器物、茶酒、游戏、职官称谓 | 按子类型扩展 ontology | B-01~06, H-06~07 |

**纵切范式**（可复用）：饮食 → 服饰 → 医药 → 诗词 → 建筑……每条线共用同一套「实体库 + 检索 + 时间线 + 事实/推论分层」。

## 五、模块架构（建议）

```
classical-novels / 红楼梦
├── 四维知识图谱          ← 总架构（见 红楼梦-知识图谱架构.md）
│   ├── ① 人物社交        ← character + relations
│   ├── ② 时空演变        ← location + event + 2D/3D 地图
│   ├── ③ 物质文化        ← dish/medicine/costume/item
│   └── ④ 文本意象        ← poem/omen/symbol + inference 边
├── 阅读器              ← 所有分支的入口（R-01~03, H-12 溯源）
├── 考证台              ← 版本学 + 脂学 + 探佚（H-10~11, 异文对勘）
├── 关系图谱            ← 人物叙事 + 时序（R-05~06, B-11~12）
├── 纵切研究            ← 百科名物，按主题分子模块
│   ├── 饮食            ← 已规划（红楼梦-饮食纵切研究.md）
│   ├── 医药
│   ├── 服饰民俗
│   ├── 建筑园林
│   └── 诗词曲赋
├── 曹学专题            ← 作者/曹家/原型对照（wiki 驱动）
└── 教学包              ← T-01~12，可从任意纵切/子图导出
```

## 六、建设优先级（红楼梦专项）

| 阶段 | 红学侧重 | 交付 |
