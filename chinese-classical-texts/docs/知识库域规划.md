# 知识库域规划

> 记录时间：2026-06-14
> 主题：殆知阁（daizhige v20，约 15694 个 txt）如何映射到本项目的知识库域划分。
> 延续 Karpathy「LLM Wiki」模式与 `classical-novels` 的 base+features schema 思路。

## 核心原则

**物理存储与逻辑域分离，按「实体模型家族」分域，按目标优先级建设。**

- **raw 层**：殆知阁原样保留（10 藏不动），只读，**永不重排** 15k 文件。
- **wiki 层**：按目标与 schema 相似度建域，指针指回 raw。**不追求覆盖全部文件**，按需建。
- 判断口诀：会互相引用 → 同域；不会 → 分开。

## 五个「实体模型家族」

按「什么 schema 装得下」归类，而非照搬 10 藏。

| 家族 | 实体模型 | 涵盖 | 规模 | 与现状 |
|------|----------|------|------|--------|
| **① 章回叙事** | 人物·回目·关系·事件 | 集藏/小说·演义·话本·宝卷；诗藏/剧曲 | ~686 | **直接复用 classical-novels schema** |
| **② 诗文集** | 作者·作品·体裁·典故·主题 | 集藏/四库别集·文总集·文评；诗藏/诗集·词集·诗话·词话·楚辞·汉赋·对联 | ~2017 | 新域，**以作者为中心** |
| **③ 史籍** | 人物·事件·时间·地点·职官 | 史藏 全部 | ~2043 | 新域 |
| **④ 义理/宗教经典** | 典籍·卷次·概念·义理（**无回目**） | 佛藏·道藏·儒藏·易藏 | ~8107 | **各自独立 schema，不可共用** |
| **⑤ 实用/博物** | 条目·分类·方法·谱录 | 医藏；子藏/兵农法算诸子笔记类书；艺藏 | ~2820 | 多为「条目型」，可只做 catalog |

> ④ 虽归一个家族便于讨论，但**佛 / 道 / 儒 / 易 必须各建独立 index + schema**，彼此不互引。

## schema 复用：1 基类 + 5 扩展

不要写 10 套互不相干的 schema，实际是一套古籍基类加选择性扩展（即 `classical-novels` 的 base+features 思路放大版）：

```
古籍基类（所有书共有）: 书名 / 作者 / 朝代 / 卷次 / 版本 / 来源路径
  ├─ +人物·回目·关系   → 章回叙事（①）
  ├─ +作品·体裁·典故   → 诗文集（②）
  ├─ +事件·时地·职官   → 史籍（③）
  ├─ +概念·义理        → 义理经典（④）
  └─ +方·药·技法·谱录  → 实用博物（⑤）
```

## 建设优先级

| 阶段 | 域 | 理由 |
|------|----|----|
| **P1** | 章回叙事 | 直接补三大名著，schema 现成，~686 本 |
| **P2** | 诗文集 / 史籍 | 体量大但 schema 清晰，为小说提供背景（清河县、宋代官制等） |
| **P3** | 实用博物 | 条目型，可只做 catalog，不必全量 ingest |
| **P4** | 佛 / 道 / 儒 / 易 | 体量最大、schema 最特殊，最后单独立项 |

> 家族 ④⑤ 中许多书**只需做目录索引（catalog）**，不必深度 ingest——这本身也是合理的分类用途。

## 与原 10 藏方案的差异（修正点）

1. raw 保持殆知阁原样不动，不按新目录重排 15k 文件。
2. 按「实体模型家族」分 5 类，而非照搬 10 藏。
3. 补回被漏掉的**四库别集（1279 本）**，并把诗藏主体（诗词）归入「诗文集」。
4. 小说 / 演义 / 话本合为一域（同 schema）；佛 / 道 / 儒 拆开（schema 不同）。
5. 用「1 基类 + 5 扩展」统一 schema。
6. 按优先级建设，不追求覆盖全部文件。
