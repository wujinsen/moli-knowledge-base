## 一、问题与目标

早期图谱页仅为 **ECharts 默认配置**（白底、灰线、无控件），观感简陋。目标：

- 读者/学者愿意「探索」而非只看静态图
- 支持推论边、矛盾边视觉区分（对接 Trust Guard / inference）
- 节点规模从 5 → 100+ 时仍有可用性能与交互

---

## 二、当前实现（V1 · 已落地）

### 2.1 技术栈

| 组件 | 选型 |
|------|------|
| 渲染 | ECharts 6 · `series.graph` · force 力导向 |
| 框架 | React 19 岛 · Astro `client:only` |
| 布局 | 专用 `GraphLayout.astro`（深色全屏，非通用 wiki 白页） |
| 主题 | `src/lib/graphTheme.ts`（阵营色 + 关系类型色） |

### 2.2 已实现交互与特效

| 能力 | 说明 |
|------|------|
| 深色沉浸背景 | 渐变 + 网格装饰 |
| 节点发光 | `shadowBlur` + 阵营配色 |
| 曲线边 | `curveness: 0.22` |
| 关系着色 | 18 种关系类型独立色 |
| 点击邻接高亮 | focus adjacency，非邻接降 opacity |
| 详情侧栏 | 选中节点关系列表 + 跳转人物页 |
| 搜索 | 匹配节点放大高亮 |
| 阵营筛选 | 底部 pill 开关 |
| 力导向 / 固定 | 一键切换 layout |
| 全屏 | Fullscreen API |
| 推论/矛盾边 | 虚线 + 琥珀色（`inference` / `contradiction`） |

### 2.3 关键文件

```
src/layouts/GraphLayout.astro      # 图谱专用深色布局
src/components/RelationGraph.tsx   # 主组件
src/lib/graphTheme.ts              # 配色
src/pages/[book]/graph.astro       # 路由
src/data/<书>.relations.json       # build_relations.py 产物
```

## 三、能力边界（诚实评估）

| 需求 | V1 ECharts | 说明 |
|------|------------|------|
| 20~80 节点探索 | ✅ 足够 | 当前栈 |
| 深色发光、筛选、邻接高亮 | ✅ 已做 | — |
| 边流动动画 | ⚠️ 有限 | graph 系列不如 lines+effect |
| 多跳路径高亮（A→…→B） | ⚠️ 需手写 | 无内置 shortest path UI |
| 100+ 节点性能 | ⚠️ 吃力 | 建议 V2 |
| 3D 球面 / 粒子网络 | ❌ | 需 Three.js / 3d-force-graph |
| 空间图谱（大观园地图） | ❌ | 独立 2D/3D 地图模块，非力导向图 |

## 四、演进路线

```
V1（现在）          V1.5              V2                 V3
ECharts 精调    →   数据充实 K0/X0  →  AntV G6      →   双视图切换
+ GraphLayout       80+ 节点           边流动/路径       社交图 + 地理图
```

| 阶段 | 交付 | 触发条件 |
|------|------|----------|
| **V1** | 沉浸式 ECharts 探索器 | ✅ 已完成 |
| **V1.5** | 红楼/西游 relations 充实 | K0 + X0 |
| **V2** | `@antv/g6` 或 `react-force-graph` | 单书节点 > 80 或需路径动画 |
| **V2** | 动态 `import()` 拆包 echarts | 首屏体积优化 |
| **V3** | 力导向 ↔ 大观园 2D 位置图 | P5 / M0 地图就绪 |

### G6 迁移保留项

- `*.relations.json` 格式不变（nodes + edges）
- `graphTheme.ts` 配色复用
- `GraphLayout.astro` 壳层复用
- inference / contradiction 边语义不变
