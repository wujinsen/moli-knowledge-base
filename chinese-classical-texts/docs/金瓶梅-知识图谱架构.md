# 金瓶梅 · 知识图谱架构（世情—政商—物价—名物）

> 记录日期：2026-06-14  
> 定位：《金瓶梅》专用图谱——**以人际勾连为经、以物质与金钱为纬**的晚明社会生态模型  
> 关联：[`金学分支与产品映射.md`](金学分支与产品映射.md) · [`产品功能规划.md`](产品功能规划.md) B-04 · [`图谱可视化方案.md`](图谱可视化方案.md)

## 一、设计原则

《金瓶梅》图谱的**复杂度与深度高于《西游记》**：不仅是神魔网络，而是可计量、可追踪的**精密社会生态**——史学家要白银流向，读者要人物纠葛，教师要 SNA 与版本差异。

```
                         ┌─────────────────────────┐
                         │   金瓶梅知识图谱          │
                         └────────────┬────────────┘
              经（人际勾连）          │          纬（物质·金钱）
     ┌───────────────────────────────┼───────────────────────────────┐
     ▼                               ▼                               ▼
  西门府社会网                  政商/利益边                    交易·名物·银价
  妻妾/帮闲/官员                贿赂/借贷/雇佣                  costume/dish/transaction
     │                               │                               │
     └───────────────────────────────┴───────────────────────────────┘
                                     ▼
              financial_event · location · variants · SNA · 白银流动图
              chapter · source · edition · Trust Guard
```

**铁律**：

- 物价、交易金额 **fact + source**；作者心理、金红递进等标 `inference: true`。
- 三版本（词话 / 崇祯 / 张竹坡）**并列**，异文进 `variants`。
- 敏感描写在 **topic 学术层**；实体层聚焦社会结构、名物、经济。
- 货币须 **原文保留 + 标准化字段**（见 §七），换算规则可修订但不可 silently 改 raw。

## 二、核心实体层（Nodes · 社会的细胞）

除人物、地点外，必须标注**商业与制度属性**（Ontology 白名单，B-04）：

### 2.1 人物（`character`）

| 维度 | 字段 / 标签 | 说明 |
|------|-------------|------|
| **社会身份** | `tags` | 官员、帮闲、商贾、妻妾、仆佣、娼妓 |
| **西门府距离** | `ximen_proximity` | `亲缘` \| `雇佣` \| `利益交换` \| `外人` |
| **正室/妾/婢** | `status` | 吴月娘 / 潘金莲 / 春梅… |
| **官职** | `office` | 理刑千户等 |
| **aliases** | 五娘、月姐… | **禁止别名分裂节点** |

```yaml
id: 应伯爵
type: character
book: 金瓶梅
tags: [帮闲]
ximen_proximity: 利益交换
faction: 帮闲圈
relations:
  - target: 西门庆
    type: 帮闲
    role: 帮闲
```

### 2.2 物质与商品（`material` · 复用 costume/dish/medicine/item）

| 子类 | `type` | 必填属性 |
|------|--------|----------|
| 珍馐美食 | `dish` | 名称、做法简述、`value`（两） |
| 名贵服饰 | `costume` | 材质、颜色、款式、`value` |
| 药材 | `medicine` | 方剂、用途 |
| 奇珍异宝 | `item` | 名称、`value`、持有者 |

**交易记录**不重复建「商品节点」时，用 **`transaction` 实体**挂价格、来源、流向（§2.3）。

### 2.3 交易记录（`transaction`）

| 属性 | 说明 |
|------|------|
| `amount` | 原文数值 |
| `currency` | 银 \| 钱 \| 贯 \| 文（原文单位） |
| `amount_normalized` | 折算为「两」的标度值（§7.1） |
| `conversion_note` | 换算依据或「约」 |
| `source` / `chapter` | 引文锚点 |
| `flow` | 来源 → 流向（buyer/seller/gift_to） |
| `item_ref` | 链接 dish/costume/character（如买丫鬟） |

### 2.4 经济事件（`financial_event`）

区别于一般 `event`，专指**可记账**的商业运作：

| 子类 | 示例 |
|------|------|
| 药铺经营 | 生药铺开张、入账 |
| 高利贷 | 放债、收息 |
| 官场贿赂 | 送礼蔡京、提刑所打点 |
| 遗产分配 | 李瓶儿家财归属 |
| 奴婢买卖 | 买春梅 |

```yaml
id: 西门庆认蔡京干爹
type: financial_event
book: 金瓶梅
subtype: 贿赂
participants: [西门庆, 蔡京]
transactions: […]          # 关联 transaction id
chapters: […]
```

### 2.5 空间与场所（`location`）

| category | 示例 |
|----------|------|
| `mansion` | 西门府、后花园、各房院落 |
| `city` | 清河县、狮子街 |
| `shop` | 生药铺、绒线铺、茶馆 |
| `temple` | 庙宇 |

边：`包含`（西门府 → 后花园）、`盘踞/经营`（西门庆 → 生药铺）。

## 四、史学与研究视角（史学家 · 教师）

### 4.1 白银流动可视化

**目标**：计算西门庆府「资金池」，展示白银经放贷、送礼、经营流向各节点。

| 能力 | 实现 |
|------|------|
| 数据源 | 全部 `transaction` + `financial_event` |
| 聚合 | 按 chapter 时间轴累加 `amount_normalized` |
| 可视化 | 桑基图 / 流向边（P3）；节点大小 ∝ 净流入 |
| 功能 ID | H-06, B-11~12, R-11（规划） |

**示例问题**：「第 20–40 回西门庆 outward 送礼总额 vs 店铺收入」

### 4.2 社会关系网络分析（SNA）

**目标**：识别帮闲圈**信息/利益掮客**（介数中心性 Betweenness Centrality）。

| 能力 | 实现 |
|------|------|
| 构建 | `/graph` 输出 `relations.json` → NetworkX（构建期或可选服务） |
| 指标 | 度中心性、介数中心性、社区发现 |
| 产品 | H-01~04 SNA 面板；与 ECharts 图谱联动高亮 |
| 典型结论 | 应伯爵、翟管家等是否为「隐形枢纽」 |

### 4.3 版本差异锚点

**目标**：词话本 vs 崇祯本（及张竹坡评本）——同事件异文、物价是否变动、情节删润。

| 能力 | 实现 |
|------|------|
| 存储 | 实体/回目 `variants[]`：`{ edition, text, amount? }` |
| UI | H-10 双栏对勘；物价字段并排 |
| 图谱 | 同节点多 edition 属性，**不建重复节点** |
| raw | `金瓶梅.txt` vs `金瓶梅词话万历本.txt` vs `editions/jinpingmei/` |

## 六、智能化工作流（六命令 · 金瓶梅专规）

### 6.1 `/ingest`：货币换算标准化

摄取每一回时，凡遇金额须 **双写**：原文 + 标准化。

**步骤（在通用 ingest 清单之上追加）**：

1. 识别货币表述：银（两）、钱/文、贯、金（少见）等。
2. 写入 `transaction` 或名物 `value` 的 `amount` + `currency`（**原文**）。
3. 调用换算规则生成 `amount_normalized`（单位：两银）。
4. `conversion_note` 注明公式或「明代某地市价约 X，存疑」。
5. 无法可靠换算 → 只存原文，`amount_normalized` 留空，`/lint` 报告。

**换算规则表（v1 · 可配置 `scripts/jinpingmei_currency.yaml`）**：

| 原文单位 | 标准化策略 | 默认系数 | 备注 |
|----------|------------|----------|------|
| 银 X 两 | `amount_normalized = X` | 1 | 直接 |
| 银 X 钱 | 1000 钱 ≈ 1 两（晚明常用口径） | ÷1000 | **因地因时浮动**，note 必填 |
| X 贯 | 1 贯 ≈ 1 两（简化）或查表 | 配置 | 与一条鞭法后白银化相关 |
| X 文 | 同「钱」 | ÷1000 | 与「钱」混用需看上下文 |
| 若干/数两 | 取区间中值或 null | — | 标「约」，guard 人工复核 |

**禁止**：为凑齐统计而无 source 地填数。

### 6.2 `/graph`：社会网 + SNA 导出

1. `build_relations.py 金瓶梅` — 合并 character relations + transaction 派生边。
2. 节点附带 `ximen_proximity`、`tags`；边附带 `chapter`。
3. **可选** `--sna`：输出 `jinpingmei.sna.json`（度、介数 Top-N）。
4. 边权重：交易金额可映射 `edge.weight` 供布局/桑基图。

### 6.3 `/query`：史学家/读者/教师

| 画像 | 典型问法 | 回答方式 |
|------|----------|----------|
| 史学家 | 全书买婢均价？贿赂链？ | 聚合 transaction + 图路径 + 出处 |
| 读者 | 西门庆有几个老婆？谁害李瓶儿？ | 人物页 + 关系子图 |
| 教师 | 帮闲圈中心是谁？ | SNA 指标 + 例句引文 |

合成结论 → 回填 `topics/金瓶梅/`（物价报告、帮闲圈分析…）。

### 6.4 `/lint`：金瓶梅专项体检

| 检查项 | 说明 |
|--------|------|
| 别名分裂 | 同一人物多个 id |
| 无 source 交易 | transaction 缺 chapter/source |
| 换算缺失 | 有 amount 无 normalized 且无 note |
| 物价版本冲突 | 词话/崇祯 amount 差异未进 variants |
| 帮闲边方向 | 帮闲边 role 是否一致 |

### 6.5 `/guard`：事实核对

- 银价、人名、官名回 `chapters/` 原文。
- 换算系数争议 → 不阻断 ingest，标 `conversion_disputed: true`。

### 6.6 `/dream`：巩固不扭曲

- 合并 aliases；补全 `ximen_proximity`。
- **不**将单次馈赠边升级为永久「巨额财富」节点属性。
- 合并重复 transaction（同回同笔）。
- 版本 variants 只并列，不 pick 赢家。
