## 一、总体架构：三层 + 领域分治

```
[ raw 原始层 ] ──ingest──▶ [ wiki 维护层 ] ──build──▶ [ app 展示层 ]
   只读真相源              LLM 生成的 md 页面          Astro 静态站
        ▲                      │   ▲
        └──── guard 回溯校验 ───┘   └── query / dream / lint
```

| 层 | 角色 | 谁维护 |
|----|------|--------|
| raw | 原始文献，只读，永不重排 | 人放入 |
| wiki | 人物/回目/主题/关系 md | LLM（受 AGENTS.md 约束）|
| app | 阅读器/关系图/检索 | 构建产物 |

**领域分治**：`chinese-classical-texts` 是「古籍」领域；将来保险/教育/IT 是同级兄弟目录，各自独立 raw/wiki/apps，互不引用。判断口诀：**会互相引用 → 同域；不会 → 分开**。

## 四、数据模型：1 基类 + 5 扩展

```
古籍基类: 书名 / 作者 / 朝代 / 卷次 / 版本 / 来源路径(raw)
  ├─ +人物·回目·关系·事件  → 章回叙事 (classical-novels)
  ├─ +作者·作品·体裁·典故  → 诗文集   (classical-poetry)
  ├─ +事件·时地·职官       → 史籍     (classical-history)
  ├─ +典籍·卷次·概念·义理  → 义理思想 (classical-canon)
  └─ +条目·方法·谱录·分类  → 工具博物 (classical-encyclopedia)
```

已实现的章回叙事实体：`character` / `monster` / `chapter` / `book` / `topic`。

- 关系走**受控词表**（亲属/社会/情感），构建期校验拦截非法值。
- 扩展模块由 `books/<书>.md` 的 `features` 开关控制：`reader` `graph` `bestiary` `items` `poems`。
- **版本异文一等公民**：`variants`（异文）+ `contradicts`（矛盾边），图谱渲染为虚线。

## 七bis、站内搜索（路线 A，已落地）

零后端的静态全文搜索，构建期生成索引，随内容自动扩展。

| 组件 | 选型 | 说明 |
|------|------|------|
| 集成 | `astro-pagefind` v2 | 挂在 `astro:build:done`，自动跑 pagefind |
| 索引器 | Pagefind 1.5 | 输出 `dist/pagefind/`，**中文分词 zh-cn 已启用** |
| UI | `@pagefind/component-ui`（`<pagefind-searchbox>`）| `configOptions={language:'zh'}` 套内置中文翻译 |
| 入口 | `/search` 页 + 头部「搜索」链接 | — |

实现要点：

- 索引区用 `data-pagefind-body` 标在回目/人物 `<article>` 上；导航等用 `data-pagefind-ignore` 排除。
- 元数据 `data-pagefind-meta="title:…"`；筛选维度 `data-pagefind-filter="书:…,类型:…"`（回目/人物/妖怪）。
- dev 模式下搜索读「上一次 `npm run build`」的索引；改内容后需重建。
- 当前仅索引已 ingest 的页（西游记示例 17 页）；金瓶梅/红楼梦正文待 `import_chapters.py` 导入后自动纳入。
- 升级方向：在此之上叠 BGE-M3 向量召回做混合检索（路线 B）。

## 七ter、古籍检索特有规则（3 个坑）

通用 RAG 清单未覆盖、但古籍必踩的三点，已定为固定工程规则：

| 坑 | 问题 | 对策 | 落地 |
|----|------|------|------|
| ① 文言文 embedding 漂移 | BGE-M3 现代语料训练，古地名/官职语义弱 | **Contextual Retrieval**：用带背景的现代摘要做检索语料 | wiki 页/回目 `summary` 已在索引区；优先用 wiki 当语料而非切 raw |
| ② 异体字/占位符污染 | `［…］〔…〕` 占位、繁简混排污染 BM25/向量 | 建索引前归一化，**raw 不动**，只在语料副本上做 | `scripts/normalize_text.py`（去占位符/全角转半角/折叠空白/可选繁→简）|
| ③ 跨版本同名 | 词话本/绣像本同回内容不同，RAG 会混答 | 检索元数据带 `edition`，异文进 `variants` | chapter schema 加 `edition`+`source`；阅读页 Pagefind 加「版本」筛选 |

铁律：**归一化、繁→简只作用于检索语料副本（`build/corpus/`），展示用原文按原书字体保留**（AGENTS 规定）。

## 十、关键技术决策与风险

| 决策 | 理由 |
|------|------|
| raw 物理移动而非 symlink | Windows symlink 兼容性差 |
| 15694 文件只编目不重排 | 保真、可溯源；ingest 按需 |
| 文件级混合体裁（剧曲/笔记）延后 | 文件夹规则切不开，留 ingest 打 `genre` |
| 静态站（Astro）而非 SSR | 内容只读为主，CDN 友好 |

**主要风险**：

- daizhige 15k 文件需章节切分才能 ingest。
- 佛藏体量大（5135）建议最后做。
- 体量增大后古籍领域可独立成 repo。
