# 西游记 · 知识图谱架构（实体—关系—属性 + 情境边）

> 记录日期：2026-06-14  
> 定位：《西游记》专用图谱——覆盖神魔网络、八十一难、地理西行、法宝克制与变身逻辑  
> 关联：[`西游学分支与产品映射.md`](西游学分支与产品映射.md) · [`图谱可视化方案.md`](图谱可视化方案.md) · [`产品功能规划.md`](产品功能规划.md) · B-03

## 一、设计原则

《西游记》与结构严谨的现代百科不同：**神话隐喻、分身/变身、跨三界地理** 并存。图谱若只建「谁认识谁」，无法回答「太上老君及其法宝制造了多少取经阻碍」这类多跳问题。

因此采用 **五层实体 + 强类型关系 + 丰富属性 + 情境边** 模型，并接入现有 `/ingest` → `/graph` 流水线：

```
                    ┌─────────────────────────┐
                    │   西游记知识图谱          │
                    └───────────┬─────────────┘
     ┌──────────┬───┴───┬───────┴───────┬──────────┬──────────┐
     ▼          ▼       ▼               ▼          ▼          ▼
  ① 角色     ② 法宝   ③ 地理         ④ 劫难     ⑤ 阵营     ⑥ 诠释
  神/妖/人   兵器     三界+洞府       事件       组织       inference
     │          │       │               │          │          │
     └──────────┴───────┴───────┬───────┴──────────┘          │
                                ▼                             │
              情境边（变身/伪装，scoped to Event）              │
              chapter · source · Trust Guard · aliases         │
                                └─────────────────────────────┘
```

**铁律**（与 [`技术方案.md`](技术方案.md)、[`AGENTS.md`](../apps/classical-novels/AGENTS.md) 一致）：

- **事实边**必须带 `source`（回目 + 原文锚点）；学者诠释（内丹对应、政治隐喻）标 `inference: true`。
- **别名不分裂节点**：孙悟空 / 行者 / 大圣 / 心猿 / 弼马温 → 同一 `id`，进 `aliases`。
- **变身不建永久边**：伪装、临时形态用 **情境边**，挂在 `event` 下，避免 `/dream` 巩固时混淆真身属性。
- raw 只读；增量写入 `content/` + `build_relations.py` → `src/data/西游记.relations.json`。

## 二、核心实体层（Nodes · Ontology 白名单）

| 类型 | `type` | 子类 / 标签 | 项目 schema | 状态 |
|------|--------|-------------|-------------|------|
| **角色** | `character` | 神仙（佛/道/天庭）、人类（皇帝/僧侣/平民） | `characters/西游记/*.md` | ✅ 部分 |
| **妖魔** | `monster` | 有背景 / 无背景；精怪（土地、山神） | 同上，`type: monster` | ✅ 5 条样本 |

## 五、特殊机制：变身与分身（情境边）

**问题**：孙悟空变假牛魔王骗芭蕉扇——若建永久边 `(孙悟空, 伪装, 铁扇公主)` 或把假牛魔王当独立节点，会与真牛魔王混淆，`/dream` 合并别名时污染属性。

**方案：情境边（Contextual Edge）**

永久图谱只保留 **稳定身份**；临时形态只在 **事件作用域** 内生效：

```yaml
# content/events/西游记/三借芭蕉扇.md
id: 三借芭蕉扇
type: event
book: 西游记
chapters: [59, 60]
locations: [火焰山]
scoped_relations:
  - subject: 孙悟空
    predicate: 伪装成
    object: 牛魔王
    chapter: 59
    source: chapters/西游记/059.md
    ephemeral: true          # 非永久边
  - subject: 孙悟空
    predicate: 获取
    object: 芭蕉扇
    via: 伪装骗得            # 因果说明
    chapter: 59
    source: …
```

**规则**：

| 规则 | 说明 |
|------|------|
| 不建假身节点 | 「假牛魔王」不作为独立 `character` |
| `ephemeral: true` | `build_relations.py` 默认 **不写入** 全局 `relations.json`，或写入单独 `ephemeral_edges` 层 |
| 可视化 | 选中 event 时叠加显示情境边；全局图只显示稳定关系 |
| `/dream` | 只 consolidate 永久边 + aliases；情境边随 event 页归档 |

## 七、与 `/ingest` · `/graph` 流水线对接

| 命令 | 西游记图谱职责 |
|------|----------------|
| **`/ingest`** | 读该回原文 → 更新/新建 character · monster · artifact · location · event → 补 **永久** `relations` → 事件内写 `scoped_relations` → 更新 `index.md` / `log.md` |
| **`/graph`** | `python scripts/build_relations.py 西游记` → 合并各实体 relations → 去重 → 校验词表 → 输出 `西游记.relations.json`（可选分层：stable / ephemeral） |
| **`/query`** | 多跳问答；复杂结论回填 `topics/西游记/` |
| **`/lint`** | 别名分裂检测（弼马温未合并）、孤儿 event、无 source 边 |
| **`/guard`** | 事实边回 raw 核对 |
| **`/dream`** | 合并 aliases、补链、重算 weight；**不**把情境边提升为永久边 |

### 7.1 `relations.json` 节点扩展（规划）

```json
{
  "book": "西游记",
  "nodes": [
    { "id": "孙悟空", "type": "character", "faction": "取经队伍", "subtype": "神仙", "weight": 100 }
  ],
  "edges": [
    { "source": "孙悟空", "target": "白骨精", "type": "降服", "chapter": 27, "inference": false }
  ],
  "meta": { "stable_only": true, "generated_at": "…" }
}
```

### 7.2 构建脚本扩展点

| 文件 | 扩展 |
|------|------|
| `content.config.ts` | 注册 `artifact`、`event` collection；扩展 `RELATION_TYPES` |
| `schema/artifact.md` | 法宝字段 |
| `schema/event.md` | event + `scoped_relations` |
| `build_relations.py` | 多 type 扫描；过滤 `ephemeral` |
| `trust_guard.py` | 新边类型校验 |
