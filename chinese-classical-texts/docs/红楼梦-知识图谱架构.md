# 红楼梦 · 知识图谱架构（四维模型）

> 记录日期：2026-06-14  
> 定位：《红楼梦》专用图谱设计——超越「谁是谁亲戚」，覆盖图谶式写作的全层隐喻  
> 关联：[`功能总览.md`](功能总览.md) · [`产品功能规划.md`](产品功能规划.md) · [`红学分支与产品映射.md`](红学分支与产品映射.md)

## 一、设计原则

《红楼梦》是**图谶式**文本：情节、结局常藏在名字、诗词、器物与谶语中。图谱若只建血缘，会丢失红学最值钱的层。

因此采用 **四个核心维度**，共享同一套「实体—关系—属性」底座，但节点类型与边语义分维扩展：

```
                    ┌─────────────────────────┐
                    │   四维红楼梦知识图谱      │
                    └───────────┬─────────────┘
          ┌─────────┬─────────┼─────────┬─────────┐
          ▼         ▼         ▼         ▼         │
     ① 人物与    ② 时空    ③ 物质与    ④ 文本意象  │
       社交网络    演变网      文化百科    与互文      │
          │         │         │         │         │
          └─────────┴────┬────┴─────────┘         │
                         ▼                         │
              共享：chapter 锚点 · edition ·       │
                    source · Trust Guard           │
                         └─────────────────────────┘
```

**铁律**（与 [`技术方案.md`](技术方案.md) 一致）：

- 事实边必须带 `source`（回目 + 原文锚点）；推论边（影射、预示）标 `inference: true`。
- raw 只读；图谱增量写入 `content/` + 构建产物 `*.relations.json` / 可选图库。

## 二、四维架构详述

### 维度 ① 人物与社交网络（基石）

**目标**：四大家族血缘 + 主仆 + 党派 + 情感盟约。

| 实体类型 | 说明 | 项目 schema |
|----------|------|-------------|
| 十二钗、公子、丫鬟、小厮 | 人物子类 | `character` + `tags` / `status` |
| 僧道、官员、亲戚 | 人物子类 | 同上 |
| 派系 | 王夫人派、邢夫人派等 | `faction`（待建）或 `character.tags` |

| 关系类型 | 示例 | 现有/扩展 |
|----------|------|-----------|
| 血缘/婚姻 | 父子、夫妻、嫡庶、姑侄 | `RELATION_TYPES` 亲属类 ✅ |
| 社会/主仆 | 宝玉→袭人（主仆） | 主仆 ✅ |
| 师生/依附 | 贾政→贾雨村 | 师徒、同僚（扩展） |
| 情感/党派 | 木石前盟、金玉良缘 | **待增**：`前盟`、`良缘`、`隶属派系` |
| 属性（非边） | 判词、曲子、花签、生辰、结局 | `character` 扩展字段 + 链到维度④ |

**人物属性挂接**：

```yaml
# character 扩展（规划）
omens: [金陵十二钗正册判词]      # → 维度④ 实体 id
flower_lot: 芙蓉                 # 花签
birthday: ...
fate: 泪尽夭亡                   # 可 inference
```

### 维度 ② 时空演变网

**目标**：荣宁府—大观园—院落的包含关系；谁住哪；何事发生于何地；随回目演变。

| 实体类型 | 示例 | 项目 schema |
|----------|------|-------------|
| 府邸 | 荣国府、宁国府 | `location` category=府邸 ✅ 已建 |
| 园林/院落 | 大观园、潇湘馆、怡红院 | `location` ✅ **28 处已录入** |
| 园外地点 | 铁槛寺、平安州、姑苏 | `location` |
| 事件 | 晴雯撕扇、螃蟹宴 | `event`（待建） |

| 关系类型 | 示例 |
|----------|------|
| 空间包含 | 大观园 → 包含 → 潇湘馆 |
| 居住 | 林黛玉 → 居住于 → 潇湘馆 |
| 发生于 | 晴雯撕扇 → 发生于 → 怡红院 |
| 出现于（时间） | 事件/人物 → 第 N 回（`valid_from`/`valid_to`） |

**与沉浸体验联动**：[`红楼梦-大观园沉浸体验.md`](红楼梦-大观园沉浸体验.md) 的 2D/3D 地图即本维度的可视化前端。

### 维度 ③ 物质与文化百科

**目标**：吃穿用度、医药、器物——清代贵族生活与「百科全书」特质。

| 实体类型 | 示例 | 项目 schema |

| 美食/茶酒 | 茄鲞、老君眉 | `dish` ✅ |
| 医药 | 冷香丸、人参养荣丸 | `medicine` ✅ |
| 器物 | 大观窑、西洋镜 | `item`（待建，`features: items`） |

| 关系类型 | 示例 |

### 维度 ④ 文本意象与互文网络（红学高阶层）

**目标**：诗词、谶语、灯谜、象征物与人物的隐喻/预示/影射——**普通小说图谱没有的一层**。

| 实体类型 | 示例 | 项目 schema |
|----------|------|-------------|
| 诗词 | 《葬花吟》 | `poem`（`features: poems`） |
| 谶语/判词 | 太虚幻境十二钗判词 | `omen` / `judgment`（待建） |
| 灯谜/酒令 | 元宵灯谜 | `riddle`（待建） |
| 象征物 | 玉、金、竹、芙蓉 | `symbol`（待建） |

| 关系类型 | 示例 | 边属性 |
|----------|------|--------|
| 创作/吟诵 | 林黛玉 → 作 → 《葬花吟》 | fact + source |
| 对应判词 | 林黛玉 → 对应 → 堪怜咏絮才 | fact |
| 隐喻/预示 | 芙蓉花签 → 隐喻 → 林黛玉（泪尽夭亡） | **inference** |
| 影射 | 晴雯 → 影射 → 林黛玉（晴为黛影） | **inference** |
| 象征 | 竹 → 象征 → 黛玉 | inference |

**路径查询示例**（红学探索）：

```
晴雯 —[影射]→ 林黛玉
芙蓉 —[象征]→ 晴雯 / 黛玉
芙蓉花签 —[隐喻]→ 林黛玉
```

系统应能沿「芙蓉」媒介自动串联 **晴雯—芙蓉—黛玉** 隐喻链（产品功能 R-10 / H-08）。

### 4.2 关系边通用字段

```yaml
relations:
  - target: 潇湘馆
    type: 居住于          # 受控词表或分维度词表
    role: 主院
    chapter: 17           # 起始回
    source: raw/daizhige/…/红楼梦.txt
    inference: false
evidence: "第17回 ……"     # 可选，guard 用
```

**关系词表扩展（规划）**——在现有亲属/社会/情感基础上增加：

```
空间: 包含  居住于  发生于  途经
物质: 持有  赠予  服用  原料包含  出现于
文本: 作  吟  对应判词  隐喻  影射  象征  预示
情感扩展: 前盟  良缘
社会扩展: 攀附  隶属派系
```

## 六、技术栈建议（双轨）

用户常提 **Neo4j + NetworkX/PyG + D3/G6**。与本项目关系如下：

| 层级 | 推荐 | 说明 |
|------|------|------|
| **真相源 + 策展** | Markdown + YAML（当前） | LLM Wiki、git 可审计、Trust Guard |
| **站点可视化** | ECharts graph（✅）+ 未来 G6 | 静态构建，无后端 |
| **多跳查询 / SNA** | NetworkX 构建期 或 Neo4j 可选服务 | 路径、中心性、社区发现 |
| **前端探索器** | ECharts → AntV G6 / D3 | 交互过滤、路径高亮 |
| **QA** | RAG + 图检索混合 | Pagefind（已有）+ 图 API |

**不建议**用 Neo4j 替代 markdown 真相层；宜作 **构建期导出或研究用副库**，主库仍为 `content/` + `raw/`。
