## 零、现状对照

| 已有 | 缺口 |
|------|------|
| 阅读器 + 320 回正文 | 悬停实体名片、3D 地图 |

## 一、核心底座（全用户共用）

### 1.1 动态本体（Ontology）引擎

| ID | 功能点 | 说明 |
|----|--------|------|
| B-01 | **书级本体配置** | 每本书独立 `ontology.yaml`：节点类型、边类型、必填属性 |
| B-02 | **红楼梦本体** | 人物、诗词、居所（大观园）、物件、节气、版本（庚辰/程甲等） |
| B-03 | **西游记本体** | 神仙、妖魔、法宝、劫难、地理位置、门派（佛/道/妖） |
| B-04 | **金瓶梅本体** | 官职、货币、服饰、饮食、曲牌、药材 |
| B-05 | **跨书基类继承** | 共用 `chapter` / `character` / `location`；各书扩展子类型 |
| B-06 | **features 开关联动** | 与 `books/*.md` 的 `features` 一致，控制 UI 模块显隐 |

### 1.2 多维属性检索

| ID | 功能点 | 说明 |
|----|--------|------|
| B-07 | **结构化实体库** | 服饰/法宝/物价等独立实体页，带数值型属性（银价、材质、兵器类型…） |
| B-08 | **复合条件查询** | 例：金瓶梅 · 服饰 · 价值 > 10 两；西游 · 水栖妖怪 · 非金属兵器 |
| B-09 | **检索 API / 查询 DSL** | 前端筛选器 + 后端（或构建期索引）支持属性过滤、AND/OR |
| B-10 | **检索结果图谱视图** | 查询结果一键投影到子图，而非仅列表 |

### 1.3 时序图谱（时间轴）

| ID | 功能点 | 说明 |
|----|--------|------|
| B-11 | **关系边时间属性** | 边带 `valid_from` / `valid_to` 或「回目区间」 |
| B-12 | **时间轴控件** | 拖动/选择回目或章节，图谱只显示该时刻有效关系 |
| B-13 | **状态演化** | 例：黛玉对宝玉态度、西门庆发迹各阶段社会网 |
| B-14 | **关键帧快照** | 预设节点（第 1/40/80/120 回）一键跳转 |

### 1.4 溯源与版本（学术底座）

| ID | 功能点 | 说明 |
|----|--------|------|
| B-15 | **三元组 → 原文锚点** | 每条关系/属性绑定 `book + chapter + 段落偏移` |
| B-16 | **版本并行图谱** | 同一关系在不同 `edition` 下的分支/缺失/改写 |
| B-17 | **Trust Guard 校验** | 图谱边写入前回 raw 核对，防 LLM 幻觉边 |

## 二、普通读者（Readers）

### 2.1 沉浸式伴读

| ID | 功能点 | 说明 |
|----|--------|------|
| R-01 | **实体识别与高亮** | 正文内人名/地名/物件自动标色 |
| R-02 | **悬停图谱名片** | 简介 + 当前回关系迷你图 + 「查看详情」 |
| R-03 | **回内上下文关系** | 名片默认按「当前阅读回」过滤，非全书静态网 |
| R-04 | **生僻注释层** | 官职/曲牌/药方等链到百科式短条目 |

### 2.2 关系可视化

| ID | 功能点 | 说明 |
|----|--------|------|
| R-05 | **四大家族树（红楼）** | 血缘 + 姻亲；主子-丫鬟从属高亮切换 |
| R-06 | **2D 力导向关系网** | 已有 ECharts 基础，需补筛选/聚类/图例 |
| R-07 | **3D 关系网（可选）** | 大规模节点时的空间布局（二期） |
| R-08 | **西游 GIS 地图** | 真实地理 + 神话地理双层；点击地点 → 妖怪/法宝/劫难卡片 |
| R-08b | **红楼 2D 大观园地图** | 全园总图、事件/纵切图层、NPC 锚点；见 [`红楼梦-大观园沉浸体验.md`](红楼梦-大观园沉浸体验.md) |
| R-09 | **取经路线动画** | 按回目顺序在地图上点亮途经点 |

### 2.3 探索型查询

| ID | 功能点 | 说明 |
|----|--------|------|
| R-10 | **最短社会路径** | 输入两人 → 关系链（如悟空→牛魔王→铁扇公主） |
| R-11 | **路径叙事模式** | 路径上每一跳可展开该回情节摘要 |
| R-12 | **「吃瓜」推荐** | 热门人物对、意外关联对（基于图算法预计算） |

## 三、史学家 / 研究者（Historians）

### 3.1 社会网络分析（SNA）

| ID | 功能点 | 说明 |
|----|--------|------|
| H-01 | **度中心性** | 谁连接最多 |
| H-02 | **中介中心性** | 「隐形枢纽」识别（金瓶梅资源流动） |
| H-03 | **社区发现** | 派系/家族/取经队伍子群自动划分 |
| H-04 | **SNA 面板** | 表格 + 图谱双向联动排序 |
| H-05 | **时段 SNA** | 与时间轴结合，对比不同阶段的中心性变化 |

### 3.2 物质文化与定量研究

| ID | 功能点 | 说明 |
|----|--------|------|
| H-06 | **主题实体汇聚** | 一键汇总：金瓶梅物价/汇率、红楼中药处方、西游法宝谱系 |
| H-07 | **统计报告导出** | Markdown / CSV / 可引用表格 |
| H-08 | **共现分析** | 人物-物件、人物-地点共现矩阵 |
| H-09 | **词频 / 称谓演变** | 同一人物在不同阶段的称呼变化（辅助发迹史研究） |

### 3.3 版本学与引文

| ID | 功能点 | 说明 |
|----|--------|------|
| H-10 | **双版本图谱对勘** | 脂评本 vs 程高本：异色边/虚线/缺失节点 |
| H-11 | **结局分歧标注** | 人物命运、关系走向的版本差异高亮 |
| H-12 | **Citation Traceback** | 点击任意边 → 跳转对应回目 + 原文段落（含 `source`/`edition`） |
| H-13 | **批量引文导出** | 选中子图 → 生成带出处的参考文献块 |

## 四、教师与学生（Teachers & Students）

### 4.1 备课与导出

| ID | 功能点 | 说明 |
|----|--------|------|
| T-01 | **子图谱圈选** | 框选/按主题筛选（诗词聚会、八十一难全景等） |

### 4.2 情节与逻辑教学

| ID | 功能点 | 说明 |
|----|--------|------|
| T-05 | **因果链图** | 例：三打白骨精前因后果链 |
| T-06 | **角色动机卡片** | 实体页增加「动机/目标/冲突」结构化字段 |
| T-07 | **情节时间线视图** | 与 B-12 共用，偏教学叙事而非 SNA |

### 4.3 测验与自测

| ID | 功能点 | 说明 |
|----|--------|------|
| T-08 | **三元组抽题** | 实体-关系-实体 → 选择/填空 |
| T-09 | **回目定位题** | 「某事件发生在第几回」 |
| T-10 | **关系判断题** | 「A 与 B 是什么关系」 |
| T-11 | **自测模式** | 随机组卷、错题回溯到原文 |
| T-12 | **题库可编辑** | 教师可审核/修正 LLM 生成的题目 |

## 六、数据前置条件

做上层功能前，需先具备：

1. **三书扩展 schema 落地**（B-02~04 对应的 Zod + wiki 模板）
2. **实体-回目-段落锚点**（B-15，否则 H-12 / T-11 无法准确）
3. **关系边带来源回目**（`source` 字段方向已有，需批量 ingest）
4. **数值型属性录入规范**（银价、材质枚举等，支撑 B-08）
5. **版本异文数据**（红楼脂评 vs 程高，支撑 H-10）

## 七、与用户画像对照

| 画像 | 核心痛点 | 主要功能块 |
|------|----------|------------|
| 普通读者 | 记不住人名、理不清关系、看不懂路线 | R-01~12 |
| 史学家 | 定量分析、物质文化提取、版本对勘 | H-01~13, B-07~17 |
| 师生 | 备课素材、考点梳理、主题化学习 | T-01~12, B-12 |

底座能力（B-01~17）为三类用户共用；`features` 开关按书启用扩展模块，避免三本书强行共用同一 UI。
