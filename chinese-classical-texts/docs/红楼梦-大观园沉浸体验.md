> 记录日期：2026-06-14（2026-06-14 修订：2D/3D 双轨并列）  
> 定位：P5 体验层——**2D 与 3D 都做**，共享同一套地点/NPC/对话数据  
> 关联：[`红学分支与产品映射.md`](红学分支与产品映射.md) · [`产品功能规划.md`](产品功能规划.md)（R-08）· [`红楼梦-饮食纵切研究.md`](红楼梦-饮食纵切研究.md)

## 一、产品愿景

用户在 **大观园** 中游览——可选 **2D 平面/等距地图** 或 **3D 漫游**——遇见贾宝玉、林黛玉、王熙凤等 NPC 并对话。

**2D 与 3D 是并列两种呈现，不是「2D 只是 3D 的降级」：**

| | 2D 地图 | 3D 世界 |
|---|---------|---------|
| **优势** | 加载快、移动端友好、一览全园、教学演示清晰 | 沉浸感、空间方位感、「走进潇湘馆」的体验 |
| **适合** | 查地点、看事件分布、美食/名物标注、课堂投影 | 探索、角色扮演感、与 NPC 近距离交互 |
| **共用** | 地点节点、NPC  roster、回目时间轴、对话 API、引文溯源 | 同左 |

核心差异于普通 AI 陪聊：

| 普通 AI 陪聊 | 本项目 NPC |
|--------------|------------|
| 人设靠 prompt 即兴发挥 | 人设来自 `characters/` wiki + 回目原文 |
| 容易编造情节 | **Trust Guard**：回答须可回溯章节/批语 |
| 与阅读、图谱割裂 | 对话中可跳转阅读器、关系图、饮食/名物条目 |
| 无时空约束 | 绑定 **地点 + 回目时间轴**，NPC 只知道「当时」该知道的 |

一句话：**2D/3D 是两种壳，同一套知识库是魂；对话是查询，不是编造。**

## 二、双轨架构（一套数据，两种视图）

```
                    ┌─────────────────────────────────┐
                    │  共享数据层（single source）      │
                    │  locations · characters · chapters │
                    │  npc_index · time_axis · dishes   │
                    └───────────────┬─────────────────┘
                                    │
              ┌─────────────────────┼─────────────────────┐
              ▼                     ▼                     ▼
     ┌────────────────┐   ┌────────────────┐   ┌────────────────┐
     │ 对话服务        │   │ 2D 视图引擎     │   │ 3D 视图引擎     │
     │ Dialogue API   │   │ Map / Canvas   │   │ R3F / Three.js │
     └────────────────┘   └────────────────┘   └────────────────┘
              │                     │                     │
              └─────────────────────┴─────────────────────┘
                                    │
                    ┌───────────────▼─────────────────┐
                    │  apps/daguanyuan/               │
                    │  /map（2D）  /world（3D）        │
                    │  顶栏一键切换 · 状态同步          │
                    └─────────────────────────────────┘
```

**工程建议**：一个独立 app `apps/daguanyuan/`，内分 `/map` 与 `/world` 路由；对话、时间轴、NPC 状态在两者间同步（同一 URL query：`?chapter=38&place=潇湘馆`）。

## 三、2D 地图（R-08 深化）

### 3.1 功能

| ID | 功能 | 说明 |
|----|------|------|
| M2D-01 | **全园总图** | 可缩放平移；分区色块：潇湘馆、怡红院、蘅芜苑、栊翠庵、稻香村、沁芳闸… |
| M2D-02 | **等距/手绘风** | 红学园林考证进 wiki，2D 用 stylized 插画风，易标注、易更新 |
| M2D-03 | **地点热点** | 点击院落 → 侧栏：住客、关联回目、名物/饮食、进入 3D |
| M2D-04 | **人物锚点** | 当前回目下 NPC 头像钉在对应院落；点击开对话 |
| M2D-05 | **事件图层** | 开关：螃蟹宴、葬花、题额… 按回目高亮路径 |
| M2D-06 | **纵切图层** | 饮食/医药/服饰等主题热力或图钉（接饮食纵切） |
| M2D-07 | **刘姥姥导览线** | 预设路线动画，对比阶层见闻 |
| M2D-08 | **跳转 3D** | 「进入此院」→ 带坐标切到 3D 并落在对应入口 |

### 3.2 技术

- SVG / Canvas / Leaflet（若叠真实 GIS 底图）+ React
- 可嵌入 `classical-novels` 的 `/honglou/map` 轻量页，或统一进 `daguanyuan`
- 首屏 < 500KB，优先于 3D 上线

## 四、3D 世界

### 4.1 功能

| ID | 功能 | 说明 |
|----|------|------|
| M3D-01 | **分区 3D 场景** | stylized 低多边形园林；与 2D 地点 ID 一一对应 |
| M3D-02 | **第一/第三人称漫游** | WASD + 鼠标；移动端虚拟摇杆 |
| M3D-03 | **区域触发** | 进入潇湘馆范围 → 环境音、侧栏回目、NPC 刷新 |
| M3D-04 | **NPC 实体** | 简模或 3D 立牌 + 名牌；点击/靠近交互 |
| M3D-05 | **跳转 2D** | 小地图 overlay 或「俯瞰全园」切 2D |
| M3D-06 | **回目时间轴** | 与 2D 共用；切换后 3D 装饰/NPC/可对话内容同步 |

### 4.2 技术

- Three.js + React Three Fiber；场景按 **院** 分块 lazy load
- 独立 chunk，不进入 Astro 主站默认 bundle
- 低端机提示「建议使用 2D 地图」，非强制降级逻辑

## 六、数据模型（2D/3D 双坐标）

```yaml
# wiki/locations/潇湘馆.md
type: location
book: 红楼梦
id: 潇湘馆
name: 潇湘馆
parent: 大观园
occupants: [林黛玉]
chapters: [17, 27, 34, ...]
features: [竹林, 窗纱, 诗稿]
coords_2d: { x: 0.42, y: 0.31 }    # 归一化 0–1，2D 总图
coords_3d: { x: 12, y: 0, z: -8 }  # 3D 场景锚点
bounds_3d: { ... }                  # 可选：进入触发 AABB
```

```yaml
# character 扩展
dialogue:
  tone: ...
  knowledge_until: 第80回
  sample_lines: [...]
  home_location: 潇湘馆
```

构建期生成 `data/daguanyuan_manifest.json`：地点列表 + 2D/3D 坐标 + 默认 NPC 布局 × 回目。

## 八、分期交付（2D / 3D 并行）

| 阶段 | 2D 交付 | 3D 交付 | 共用 |
|------|---------|---------|------|
| **M0** | 可点总图 + 5 院 + 侧栏 + 静态台词 | — | location 数据 + 对话 MVP |
| **M1** | 回目轴 + NPC 钉点 + 事件图层 | 潇湘馆单院漫游 + 黛玉 NPC | RAG 对话 + 引文跳转 |
| **M2** | 全园 15 院 + 纵切图层 + 刘姥姥线 | 全园分区 + 10 主 NPC | 2D↔3D 切换同步 |
| **M3** | 螃蟹宴/葬花等事件态 | 同事件 3D 布置 | 饮食/教师模式 |
| **M4** | 导出教学用 PNG/SVG | 语音、UGC 导览 | 多人同游（可选） |

**并行原则**：M0–M1 **2D 先行**（验证数据、对话闭环）；M1 起 **3D 与 2D 同迭代**，不再把 2D 当临时方案。

对应总路线 **P5**；图谱数据模型见 [`红楼梦-知识图谱架构.md`](红楼梦-知识图谱架构.md) 维度②。

## 九、风险与原则

1. **数据先行**：无 location/character，2D 3D 都是空壳。
2. **对话一致**：同一 NPC 在 2D 点击与 3D 走近，回答逻辑完全相同。
3. **幻觉**：Trust Guard + 教师模式，两种视图均适用。
4. **性能分轨**：2D 进主站轻路由；3D 独立 app/chunk。
5. **scope**：分区可识别即可；红学考证更新 wiki，2D/3D 视图跟随 manifest 重建。
